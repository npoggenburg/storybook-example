## Was bringt uns das ganze?

* Das gesamte Frontend wird unabhängig vom CMS gebaut und kann z. B. für andere Projekte und andere CMS (Wir bauen gerade inen Wordpress Blog..) genutzt werden.
* Dezentralisierung: Frontend Entwikler bauen Frontend, Integratoren integrieren, Devs entwickeln
* Grafiker können dank mdx ein Changelog führen und müssen maximal git pull/add/commit/push + Markup kennne
* Es ist sofort ersichtlich, welche Komponenten kein, oder ein veraltetes Layout im Design Tool haben.
* Für kleinere Projekte eher overkill.
* Die Integration von Tailwind ermöglicht copy/paste in alle Projekte, die auf diesem Storybook basieren.
* Kleine Anpassungen werden aufwändiger. Da ggf. die benötigte CSS Klasse nicht im CSS File ist.
